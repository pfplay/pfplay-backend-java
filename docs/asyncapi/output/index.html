PFPlay WebSocket API 2.0.0
  • Private
  • application/json

PFPlay 실시간 음악 파티 플랫폼의 WebSocket 이벤트 규격서입니다.

연결 방식

  • 프로토콜: STOMP over WebSocket
  • 엔드포인트: ws://{host}/ws
  • 인증: JWT 토큰 (쿠키 기반, 핸드셰이크 시 검증)

메시지 흐름

Client ──→ /pub/...          (클라이언트 → 서버)
Server ──→ /sub/...          (서버 → 클라이언트, 그룹 브로드캐스트)
Server ──→ /user/.../sub/... (서버 → 클라이언트, 개인 메시지)

공통 규칙

  • 모든 서버→클라이언트 브로드캐스트 메시지에는 공통 메타데이터가 포함됩니다:
    • partyroomId (integer) — 파티룸 식별자 (플랫 정수)
    • eventType (string) — 이벤트 종류 (과거형 snake_case)
    • id (string) — 메시지 고유 ID (UUID v4)
    • timestamp (integer) — 메시지 발행 시각 (Unix epoch ms)
  • 이벤트 이름은 과거형을 사용합니다 (예: playback_started, crew_entered).
